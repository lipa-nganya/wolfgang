API Documentation

DeliveryOS Valkyrie API

The DeliveryOS Valkyrie API allows you to programmatically create orders, manage drivers, and track deliveries. The API follows RESTful principles and returns JSON responses.

Base URLs:
Production: https://api.deliveryos.com/api/valkyrie/v1
Sandbox: http://localhost:5001/api/valkyrie/v1

Sandbox vs Production

Sandbox
  • Free to use
  • Mock drivers and data
  • Rate limited (100 requests/hour)
  • Auto-expires after 90 days
Production
  • Requires approval
  • Real drivers and deliveries
  • Higher rate limits
  • Billing enabled

Authentication

All API requests require authentication using an API key. Include your API key in the Authorization header as a Bearer token.

Authorization: Bearer sk_test_your_api_key_here # Example with cURL curl -X GET "http://localhost:5001/api/valkyrie/v1/orders" \\ -H "Authorization: Bearer sk_test_your_api_key_here"

API Key Format

Sandbox keys: Start with sk_test_

Production keys: Start with sk_live_

Create Order

POST /orders

POST http://localhost:5001/api/valkyrie/v1/orders Content-Type: application/json Authorization: Bearer sk_test_your_api_key { "customerName": "John Doe", "customerPhone": "+254700000000", "deliveryAddress": "123 Main Street, Nairobi", "items": [ { "name": "Product Name", "quantity": 2, "price": 500 } ], "notes": "Handle with care" }

Get Order Status

GET /orders/:orderId

GET http://localhost:5001/api/valkyrie/v1/orders/123 Authorization: Bearer sk_test_your_api_key

Riders (Drivers)

The Valkyrie API provides access to riders (drivers) for order fulfillment. Riders can be assigned to orders and tracked in real-time.

Get Available Riders

GET /riders

Retrieve a list of available riders for your partner account.

GET http://localhost:5001/api/valkyrie/v1/riders Authorization: Bearer sk_test_your_api_key

Response Format

{ "success": true, "riders": [ { "id": 1, "name": "John Driver", "phoneNumber": "+254700000000", "status": "active", "vehicleType": "motorcycle" } ] }

Rider Status

Riders can have the following statuses:

  • active - Available for new orders
  • on_delivery - Currently delivering an order
  • offline - Not available

Assign Rider to Order

Riders are automatically assigned to orders based on availability and location. You can also manually assign a rider when creating an order.

Note: In sandbox mode, riders are mock/test data. Production mode uses real riders from the DeliveryOS network.

Error Handling

The API uses standard HTTP status codes. Errors are returned in JSON format.

Error Response Format

{ "error": "Bad Request", "message": "Invalid delivery address", "code": "INVALID_ADDRESS" }

Status Codes

200 - Success

201 - Created

400 - Bad Request

401 - Unauthorized

403 - Forbidden

404 - Not Found

429 - Rate Limited

500 - Server Error
